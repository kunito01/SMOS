## Обзор

Studio Map OS объединяет бренды, группы проектов, проекты, участников, программное обеспечение, расходы, временные шкалы, контрольные точки выпусков и архивы в одном визуальном рабочем пространстве. Система помогает независимым авторам вести несколько параллельных проектов, не сводя творческий процесс к обычному списку задач.

Текущая версия представляет собой устанавливаемое Local-First-PWA. Деловые данные остаются на устройстве, шифруются с помощью Web Crypto и сохраняются в IndexedDB. Интегрированы Web App Manifest, Service Worker, резервная офлайн-страница, значки приложения и процесс автономной упаковки. Локальные учётные записи, ключи восстановления и резервные копии обрабатываются в браузере. Вход через Apple / iCloud позволяет по выбору синхронизировать зашифрованные данные рабочего пространства через приватную базу CloudKit пользователя; собственный удалённый бизнес-бэкенд и управляемая приложением серверная аутентификация не входят в проект.

## Что нового в V 1.2

- **Два независимых варианта учётной записи** — продолжайте использовать полностью локальную учётную запись IndexedDB или отдельно входите через Apple / iCloud; эти варианты не зависят друг от друга.
- **Зашифрованная синхронизация CloudKit** — сохраняйте зашифрованную копию на устройстве и синхронизируйте с приватной базой CloudKit только шифротекст, используя привязку учётной записи, блокировки записи, проверку change tag и явное разрешение конфликтов.
- **Более безопасное восстановление устройства** — используйте однократно показываемый 16-значный ключ восстановления, привязанный к устройству неэкспортируемый ключ Web Crypto и зашифрованные резервные копии устройства, рабочего пространства и проекта.
- **Визуальный редактор рабочих процессов** — создавайте повторно используемые схемы со связанными узлами, цветами, вложениями, масштабированием, связями с проектами, автосохранением и автономной публикацией в HTML.
- **Расширенная передача проекта** — связывайте процессы с проектами, управляйте этапами Demo и официального релиза, платежами и временными шкалами, а также экспортируйте общие HTML-отчёты.
- **Связанные библиотеки затрат** — повторно используйте и синхронизируйте шаблоны затрат на персонал и ПО, рассчитывайте бюджеты этапов с учётом распределения и получайте напоминания об оплате подписок.
- **Более безопасное редактирование и навигация** — защита несохранённых изменений, отдельные подтверждения действий, улучшенное управление архивом и хранилищем, а также понятные состояния сохранения, удаления, выхода и восстановления.
- **PWA и международный интерфейс** — улучшены GitHub Pages/PWA, навигация и окна, сохранены одиннадцать рабочих языков и добавлен двенадцатый развлекательный вариант на шумерской клинописи.

## Основные функции

- **Бренды и группы проектов** — создавайте отдельные бренды и организуйте работу с помощью повторно используемых типов групп проектов.
- **Рабочие пространства проектов** — отслеживайте статусы, этапы, цели, задачи, участников, инструменты, материалы, версии и журнал действий.
- **Визуальные временные шкалы** — настраивайте даты этапов, задачи, ответственных, инструменты, заметки и пользовательские строки для каждого проекта.
- **Структурированные бюджеты** — планируйте по этапам затраты на персонал, поездки, повседневные расходы, аутсорсинг, дополнительные расходы и программное обеспечение, включая налоги и резерв на непредвиденные обстоятельства.
- **Расходы и дебиторская задолженность** — объединяйте бюджеты, фактические расходы, подписки на ПО и графики платежей по проектам.
- **Повторно используемые библиотеки** — ведите списки участников, программных инструментов, подписок и шаблонов затрат.
- **Архивирование и переносимость** — архивируйте проекты, экспортируйте отдельный проект или создавайте резервную копию всех данных Studio Map OS в браузере.
- **Доступ только для чтения** — выбирайте, будут ли снимки проектов включать временные шкалы, результаты работы, участников, инструменты, материалы, версии и предварительный просмотр расходов.
- **Международный интерфейс** — используйте английский, упрощённый китайский, японский, испанский, португальский, немецкий, французский, русский, турецкий, корейский, тайский или явно неакадемический развлекательный вариант шумерской клинописи.

## Поддержка PWA

Studio Map OS содержит полноценную структуру интеграции PWA:

- Web App Manifest с режимом отображения `standalone` и начальным URL `/login`.
- Значки 192×192 и 512×512, маскируемые значки и Apple Touch Icons.
- Автоматическая регистрация Service Worker и кеширование во время выполнения на базе Serwist.
- Предварительное кеширование корневой страницы, страниц входа, регистрации и офлайн-режима, манифеста, ресурса бренда и значков PWA.
- Резервная страница навигации по документам по адресу `/offline`.
- Метаданные для домашнего экрана iOS, цвета темы и `viewport-fit=cover`.
- Переносимый пакет PWA, содержащий автономный сервер Next.js, статические ресурсы и сценарий запуска.

> [!NOTE]
> В режиме разработки Service Worker отключён, чтобы устаревший кеш не мешал разработке. Проверяйте установку, кеширование и работу без сети в производственной сборке на `localhost` или по HTTPS.

## Начало работы

### Требования

- Рекомендуется Node.js 20 LTS
- npm
- Современный браузер с поддержкой Web Crypto и IndexedDB

### Установка и запуск

```bash
git clone https://github.com/kunito01/SMOS.git
cd SMOS
npm install
npm run dev
```

Откройте [http://localhost:3000/register](http://localhost:3000/register), чтобы создать первую локальную учётную запись.

При первом использовании:

1. Укажите имя, адрес электронной почты и пароль длиной не менее восьми символов.
2. Создайте новое рабочее пространство.
3. Сразу скопируйте или скачайте созданный 16-значный ключ восстановления рабочего пространства.
4. Прежде чем войти в рабочее пространство, подтвердите, что ключ восстановления надёжно сохранён.

> [!IMPORTANT]
> Ключ восстановления не хранится вместе с учётной записью в открытом виде. Если потеряны и пароль, и ключ восстановления, а пригодной резервной копии не осталось, данные рабочего пространства могут оказаться невосстановимыми.

В существующие локальные учётные записи можно войти по адресу [http://localhost:3000/login](http://localhost:3000/login). Предварительно настроенной учётной записи, принимающей произвольный пароль, не существует.

### Производственный режим и проверка PWA

```bash
npm run build
npm run start
```

Откройте [http://localhost:3000/login](http://localhost:3000/login) в браузере с поддержкой PWA, чтобы проверить Manifest, Service Worker и точку входа для установки. Браузеры считают `localhost` безопасным контекстом; в производственных развёртываниях следует использовать HTTPS.

### Создание переносимого пакета PWA

```bash
npm run package:pwa
```

Пакет записывается в `output/pwa/studio-map-os-pwa/`. Он содержит автономный сервер, ресурсы PWA и сценарии запуска для Windows (`START_STUDIO_MAP_OS.bat`), macOS (`START_STUDIO_MAP_OS.command`) и терминалов Linux/macOS (`START_STUDIO_MAP_OS.sh`). Все средства запуска по умолчанию используют `127.0.0.1:3002`.

## Модель данных и безопасности

```text
Страницы React
    ↓
Локальные адаптеры в lib/api
    ↓
Деловая база данных в памяти
    ↓
Шифрование Web Crypto
    ↓
Хранение в IndexedDB
```

- Деловые данные изолированы по рабочим пространствам и хранятся в виде зашифрованных записей IndexedDB.
- Пароль разблокирует защищённый мастер-ключ рабочего пространства; после входа мастер-ключ используется только в памяти.
- 16-значный ключ восстановления позволяет восстановить мастер-ключ рабочего пространства и разблокировать зашифрованные файлы резервных копий.
- Записи рабочих пространств и контейнеры резервных копий используют браузерную криптографию, включая PBKDF2, HKDF и AES-GCM.
- Полная резервная копия устройства содержит локальные учётные записи, рабочие пространства, настройки и зашифрованные снимки базы данных. Экспорт рабочих пространств и проектов также зашифрован.
- Браузеры могут отклонять запросы на постоянное хранение, поэтому зашифрованные резервные копии остаются важнейшей частью защиты данных.

> [!WARNING]
> Эти механизмы не проходили независимый аудит безопасности. Они не заменяют профессиональное управление ключами, серверные резервные копии или корпоративные системы идентификации.

## Расчёты в нескольких валютах

В настоящее время для расчётов и отображения поддерживаются следующие валюты:

- CNY — китайский юань
- USD — доллар США
- JPY — японская иена
- EUR — евро

Браузер напрямую получает справочные курсы через сервис Frankfurter, использующий данные ЕЦБ, а при сбое обращается к кешу браузера или встроенным курсам. Курсы валют предназначены для внутренних оценок студии, а не для расчётов или финансовых рекомендаций.

## Файлы резервных копий

| Тип | Содержимое | Типичное имя файла |
| --- | --- | --- |
| Полная резервная копия устройства | Все локальные учётные записи, рабочие пространства, настройки и зашифрованные данные | `studio-map-os-*.smos-backup.json` |
| Резервная копия рабочего пространства | Деловые данные и настройки текущего рабочего пространства | `studio-map-os-workspace-*.smos-backup.json` |

Перед восстановлением проверьте тип резервной копии и ключ восстановления. Полное восстановление устройства может заменить существующие данные Studio Map OS в текущем браузере.

## Текущие границы публичного доступа

Снимки для публичного доступа только для чтения пока остаются в браузере и в пределах origin сайта, где они были созданы. Ссылку общего доступа можно открыть локально, но данные не публикуются автоматически на удалённом сервере. Поэтому:

- Ссылка может перестать работать в другом браузере, после очистки данных сайта или на другом устройстве.
- Эта возможность пока не равноценна общедоступной странице, размещённой в интернете.
- Для обмена между устройствами потребуются удалённое хранилище, контроль доступа и инфраструктура отзыва прав.

## Интернационализация

Интерфейс предлагает двенадцать языковых вариантов. Шумерская клинопись — намеренно игровая псевдолокализация: опасные действия сохраняют английскую подсказку, а даты, числа, валюты и символы используют тот же формат и обычный латинский шрифт, что и английский вариант. Она не предназначена для научного использования. Остальные локали используют английский вариант, если отдельный ключ отсутствует.

Репозиторий пока не содержит автоматизированных модульных или сквозных тестов. Изменения в шифровании, миграции, восстановлении или расчётах бюджета следует дополнительно проверять перед слиянием.

## Текущие ограничения

- Бизнес-API пока реализованы как локальные адаптеры браузера; производственный сервер не подключён.
- Новые проекты наследуют части структуры демонстрационного проекта, а не создаются из полностью пустого шаблона.
- Сценарии редактирования фактических расходов, материалов и журнала действий реализованы не полностью.
- Ещё необходимо подключить сведения о группах проектов, отзыв общего доступа и управление сроком действия ссылок.
- После полной перезагрузки страницы рабочее пространство необходимо снова разблокировать паролем.
- Поддержка PWA интегрирована, но автоматизированные тесты Lighthouse, процесса установки и сквозной работы без сети пока не настроены.
- Некешированные динамические страницы и активные сетевые конечные точки могут быть недоступны без сети; резервная офлайн-страница и локальные данные не заменяют удалённые API.
