## ภาพรวม

Studio Map OS เชื่อมโยงแบรนด์ กลุ่มโครงการ โครงการ บุคลากร ซอฟต์แวร์ ต้นทุน ไทม์ไลน์ จุดตรวจสอบการเผยแพร่ และคลังเก็บไว้ในพื้นที่ทำงานเชิงภาพแห่งเดียว ระบบช่วยให้ครีเอเตอร์อิสระบริหารหลายโครงการพร้อมกันได้ โดยไม่ลดทอนกระบวนการสร้างสรรค์ให้เหลือเพียงรายการงานทั่วไป

เวอร์ชันปัจจุบันเป็น PWA แบบให้ความสำคัญกับข้อมูลในเครื่องที่สามารถติดตั้งได้ ข้อมูลทางธุรกิจจะอยู่ในอุปกรณ์ เข้ารหัสด้วย Web Crypto และจัดเก็บถาวรใน IndexedDB ระบบได้รวม Web App Manifest, Service Worker, หน้าสำรองเมื่อออฟไลน์ ไอคอนแอปพลิเคชัน และขั้นตอนการแพ็กเกจแบบสแตนด์อโลนไว้แล้ว บัญชีในเครื่อง คีย์กู้คืน และข้อมูลสำรองได้รับการจัดการในเบราว์เซอร์ การเข้าสู่ระบบด้วย Apple / iCloud สามารถเลือกซิงค์ข้อมูลพื้นที่ทำงานที่เข้ารหัสผ่านฐานข้อมูล CloudKit ส่วนตัวของผู้ใช้ได้ โดยโครงการนี้ไม่มีแบ็กเอนด์ธุรกิจระยะไกลแบบกำหนดเองหรือการยืนยันตัวตนบนเซิร์ฟเวอร์ที่แอปจัดการ

## มีอะไรใหม่ใน V 1.2

- **บัญชีสองเส้นทางที่แยกจากกัน** — ใช้บัญชี IndexedDB ที่อยู่ในเครื่องทั้งหมดต่อไป หรือเข้าสู่ระบบด้วย Apple / iCloud แยกต่างหาก โดยทั้งสองเส้นทางไม่พึ่งพากัน
- **การซิงค์ CloudKit แบบเข้ารหัส** — เก็บสำเนาที่เข้ารหัสไว้ในอุปกรณ์และซิงค์เฉพาะข้อความเข้ารหัสกับฐานข้อมูล CloudKit ส่วนตัว พร้อมการผูกบัญชี ล็อกการเขียน ตรวจสอบ change tag และแก้ข้อขัดแย้งระหว่างเครื่องกับคลาวด์อย่างชัดเจน
- **การกู้คืนอุปกรณ์ที่ปลอดภัยขึ้น** — ใช้คีย์กู้คืน 16 หลักที่แสดงเพียงครั้งเดียว คีย์ Web Crypto ที่ผูกกับอุปกรณ์และส่งออกไม่ได้ รวมถึงข้อมูลสำรองแบบเข้ารหัสทั้งอุปกรณ์ พื้นที่ทำงาน และโครงการ
- **ตัวสร้างเวิร์กโฟลว์เชิงภาพ** — สร้างบอร์ดที่นำกลับมาใช้ได้ พร้อมโหนดเชื่อมต่อ สี ไฟล์แนบ การซูม การเชื่อมโยงโครงการ บันทึกอัตโนมัติ และแชร์เป็น HTML แบบสแตนด์อโลน
- **การส่งมอบโครงการที่ครบขึ้น** — เชื่อมเวิร์กโฟลว์กับโครงการ จัดการจุด Demo และเผยแพร่อย่างเป็นทางการ การชำระเงินและไทม์ไลน์ พร้อมส่งออกรายงานโครงการ HTML สำหรับแชร์
- **ไลบรารีต้นทุนที่เชื่อมโยงกัน** — ใช้และซิงค์แม่แบบต้นทุนบุคลากร/ซอฟต์แวร์ คำนวณงบประมาณตามสัดส่วนการจัดสรร และรับการแจ้งเตือนชำระค่าสมัครใช้บริการ
- **การแก้ไขและนำทางที่ปลอดภัยขึ้น** — ป้องกันการเปลี่ยนแปลงที่ยังไม่บันทึก ยืนยันการทำงานเฉพาะจุด ปรับปรุงการควบคุมคลังเก็บและพื้นที่จัดเก็บ พร้อมสถานะบันทึก ลบ ออกจากระบบ และกู้คืนที่ชัดเจน
- **PWA และอินเทอร์เฟซหลายภาษา** — ปรับปรุง GitHub Pages/PWA การนำทางและโมดอล คง 11 ภาษาหลัก และเพิ่มตัวเลือกอักษรลิ่มสุเมเรียนเพื่อความบันเทิงเป็นภาษาที่ 12

## คุณสมบัติหลัก

- **แบรนด์และกลุ่มโครงการ** — สร้างแบรนด์แยกจากกันอย่างชัดเจน และจัดระเบียบงานด้วยประเภทกลุ่มโครงการที่นำกลับมาใช้ได้
- **พื้นที่ทำงานของโครงการ** — ติดตามสถานะ ขั้นตอน เป้าหมาย งาน บุคลากร เครื่องมือ สื่อ เวอร์ชัน และบันทึกกิจกรรม
- **ไทม์ไลน์เชิงภาพ** — กำหนดวันที่ของแต่ละขั้นตอน งาน ผู้รับผิดชอบ เครื่องมือ บันทึก และแถวแบบกำหนดเองสำหรับแต่ละโครงการ
- **งบประมาณแบบมีโครงสร้าง** — วางแผนค่าบุคลากร ค่าเดินทาง ค่าใช้จ่ายรายวัน งานจ้างภายนอก ต้นทุนเพิ่มเติม และซอฟต์แวร์ตามขั้นตอน รวมถึงภาษีและเงินสำรอง
- **ต้นทุนและลูกหนี้** — รวมงบประมาณ ต้นทุนจริง การสมัครใช้ซอฟต์แวร์ และกำหนดการรับชำระเงินของโครงการ
- **ไลบรารีที่นำกลับมาใช้ได้** — ดูแลบุคลากร เครื่องมือซอฟต์แวร์ การสมัครใช้บริการ และแม่แบบต้นทุน
- **คลังเก็บและการเคลื่อนย้ายข้อมูล** — เก็บโครงการถาวร ส่งออกโครงการแต่ละรายการ หรือสำรองข้อมูล Studio Map OS ทั้งหมดในเบราว์เซอร์
- **การแชร์แบบอ่านอย่างเดียว** — เลือกว่าจะให้สแนปช็อตโครงการมีไทม์ไลน์ ผลงานส่งมอบ บุคลากร เครื่องมือ สื่อ เวอร์ชัน และตัวอย่างต้นทุนหรือไม่
- **อินเทอร์เฟซหลายภาษา** — ใช้งานได้ในภาษาอังกฤษ จีนตัวย่อ ญี่ปุ่น สเปน โปรตุเกส เยอรมัน ฝรั่งเศส รัสเซีย ตุรกี เกาหลี ไทย หรือตัวเลือกอักษรลิ่มสุเมเรียนเพื่อความบันเทิงที่ระบุชัดว่าไม่ใช่เชิงวิชาการ

## การรองรับ PWA

Studio Map OS มีโครงสร้างการผสานรวม PWA ที่ครบถ้วน ดังนี้

- Web App Manifest ที่ใช้โหมดแสดงผล `standalone` และกำหนด `/login` เป็น URL เริ่มต้น
- ไอคอนขนาด 192×192, 512×512 แบบ maskable และ Apple Touch
- การลงทะเบียน Service Worker อัตโนมัติและการแคชขณะทำงานด้วย Serwist
- การแคชล่วงหน้าสำหรับหน้าหลัก หน้าเข้าสู่ระบบ หน้าลงทะเบียน หน้าออฟไลน์ Manifest สินทรัพย์แบรนด์ และไอคอน PWA
- หน้าสำรองสำหรับการนำทางเอกสารที่ `/offline`
- เมทาดาทาสำหรับหน้าจอโฮมของ iOS สีธีม และ `viewport-fit=cover`
- ชุด PWA แบบพกพาที่ประกอบด้วยเซิร์ฟเวอร์ Next.js แบบสแตนด์อโลน สินทรัพย์แบบคงที่ และสคริปต์เริ่มต้น

> [!NOTE]
> โหมดพัฒนาจะปิดใช้งาน Service Worker เพื่อป้องกันไม่ให้แคชเก่ารบกวนการพัฒนา การยืนยันตัวตน Apple ID ในเครื่องต้องใช้ต้นทาง HTTPS ที่ตรงกันคือ [https://localhost:3305](https://localhost:3305) เท่านั้น ไม่รองรับ HTTP หรือพอร์ตในเครื่องอื่น

## เริ่มต้นใช้งาน

### ความต้องการของระบบ

- แนะนำ Node.js 20 LTS
- npm
- เบราว์เซอร์สมัยใหม่ที่รองรับ Web Crypto และ IndexedDB

### ติดตั้งและเรียกใช้

```bash
git clone https://github.com/kunito01/SMOS.git
cd SMOS
npm install
npm run dev -- --port 3305
```

เปิด [https://localhost:3305/register](https://localhost:3305/register) เพื่อสร้างบัญชีในเครื่องบัญชีแรก

เมื่อใช้งานครั้งแรก:

1. กรอกชื่อ ที่อยู่อีเมล และรหัสผ่านอย่างน้อย 8 อักขระ
2. สร้างพื้นที่ทำงานใหม่
3. คัดลอกหรือดาวน์โหลดคีย์กู้คืนพื้นที่ทำงาน 16 หลักที่ระบบสร้างขึ้นทันที
4. ยืนยันว่าได้จัดเก็บคีย์กู้คืนไว้อย่างปลอดภัยก่อนเข้าสู่พื้นที่ทำงาน

> [!IMPORTANT]
> คีย์กู้คืนจะไม่ถูกจัดเก็บเป็นข้อความธรรมดาร่วมกับบัญชี หากรหัสผ่านและคีย์กู้คืนสูญหายทั้งคู่ และไม่มีข้อมูลสำรองที่ใช้ได้เหลืออยู่ ข้อมูลพื้นที่ทำงานอาจไม่สามารถกู้คืนได้

บัญชีในเครื่องที่มีอยู่สามารถเข้าสู่ระบบได้ที่ [https://localhost:3305/login](https://localhost:3305/login) ไม่มีบัญชีที่กำหนดไว้ล่วงหน้าซึ่งยอมรับรหัสผ่านใดก็ได้

### โหมดโปรดักชันและการตรวจสอบ PWA

```bash
npm run build
```

ตรวจสอบ PWA สำหรับใช้งานจริงผ่านเว็บไซต์ HTTPS ที่เผยแพร่แล้ว สำหรับการยืนยันตัวตน Apple ID ในเครื่อง ให้ใช้เฉพาะเซิร์ฟเวอร์พัฒนา HTTPS ด้านบนและเปิด [https://localhost:3305/login](https://localhost:3305/login) เท่านั้น ไม่รองรับ HTTP หรือพอร์ตในเครื่องอื่น

### สร้างชุด PWA แบบพกพา

```bash
npm run package:pwa
```

ชุดไฟล์จะถูกเขียนไปที่ `output/pwa/studio-map-os-pwa/` ภายในประกอบด้วยเซิร์ฟเวอร์แบบสแตนด์อโลน สินทรัพย์ PWA และสคริปต์เริ่มต้นสำหรับ Windows (`START_STUDIO_MAP_OS.bat`), macOS (`START_STUDIO_MAP_OS.command`) และเทอร์มินัล Linux/macOS (`START_STUDIO_MAP_OS.sh`) ตัวเรียกใช้เหล่านี้ไม่สามารถสร้างต้นทาง HTTPS ที่รองรับ Apple ID ได้ด้วยตัวเอง การยืนยันตัวตน Apple ID ในเครื่องต้องใช้ [https://localhost:3305](https://localhost:3305)

## แบบจำลองข้อมูลและความปลอดภัย

```text
หน้า React
    ↓
อะแดปเตอร์ในเครื่องที่ lib/api
    ↓
ฐานข้อมูลทางธุรกิจในหน่วยความจำ
    ↓
การเข้ารหัสด้วย Web Crypto
    ↓
การจัดเก็บถาวรใน IndexedDB
```

- ข้อมูลทางธุรกิจถูกแยกตามพื้นที่ทำงานและจัดเก็บเป็นระเบียน IndexedDB ที่เข้ารหัสแล้ว
- รหัสผ่านใช้ปลดล็อกมาสเตอร์คีย์ของพื้นที่ทำงานที่ได้รับการป้องกัน โดยมาสเตอร์คีย์จะถูกใช้เฉพาะในหน่วยความจำหลังจากเข้าสู่ระบบ
- คีย์กู้คืน 16 หลักสามารถใช้กู้คืนมาสเตอร์คีย์ของพื้นที่ทำงานและปลดล็อกไฟล์สำรองที่เข้ารหัส
- ระเบียนพื้นที่ทำงานและซองข้อมูลสำรองใช้การเข้ารหัสของเบราว์เซอร์ ซึ่งรวมถึง PBKDF2, HKDF และ AES-GCM
- ข้อมูลสำรองทั้งอุปกรณ์ประกอบด้วยบัญชีในเครื่อง พื้นที่ทำงาน การตั้งค่า และสแนปช็อตฐานข้อมูลที่เข้ารหัส การส่งออกพื้นที่ทำงานและโครงการก็ได้รับการเข้ารหัสเช่นกัน
- เบราว์เซอร์อาจปฏิเสธคำขอพื้นที่จัดเก็บถาวร ดังนั้นข้อมูลสำรองที่เข้ารหัสจึงยังเป็นส่วนสำคัญของการปกป้องข้อมูล

> [!WARNING]
> กลไกเหล่านี้ยังไม่ผ่านการตรวจสอบความปลอดภัยโดยหน่วยงานอิสระ และไม่สามารถใช้แทนการจัดการคีย์โดยผู้เชี่ยวชาญ การสำรองข้อมูลบนเซิร์ฟเวอร์ หรือระบบข้อมูลประจำตัวระดับองค์กรได้

## ขอบเขตการแชร์สาธารณะในปัจจุบัน

ขณะนี้ระเบียนสำหรับแชร์แบบอ่านอย่างเดียวยังคงอยู่ในเบราว์เซอร์และต้นทางเว็บไซต์ที่สร้างระเบียนนั้น URL สำหรับแชร์สามารถเปิดในเครื่องได้ แต่ข้อมูลจะไม่ถูกเผยแพร่ไปยังเซิร์ฟเวอร์ระยะไกลโดยอัตโนมัติ ดังนั้น:

- ลิงก์อาจหยุดทำงานเมื่อเปิดในเบราว์เซอร์อื่น หลังล้างข้อมูลเว็บไซต์ หรือบนอุปกรณ์อื่น
- ความสามารถนี้ยังไม่เทียบเท่าหน้าสาธารณะที่โฮสต์บนอินเทอร์เน็ต
- การแชร์ข้ามอุปกรณ์จะต้องมีพื้นที่จัดเก็บระยะไกล การควบคุมการเข้าถึง และโครงสร้างพื้นฐานสำหรับเพิกถอนสิทธิ์

## การรองรับหลายภาษา

อินเทอร์เฟซมีตัวเลือกภาษา 12 แบบ อักษรลิ่มสุเมเรียนเป็นคำแปลจำลองเพื่อความสนุกโดยตั้งใจ ข้อความที่เกี่ยวกับความปลอดภัยจะคงคำแนะนำภาษาอังกฤษ ส่วนวันที่ ตัวเลข สกุลเงิน และสัญลักษณ์ใช้รูปแบบและฟอนต์ละตินปกติแบบเดียวกับตัวเลือกภาษาอังกฤษ ไม่ได้มีไว้ใช้เชิงวิชาการ ภาษาอื่นจะย้อนกลับไปใช้ภาษาอังกฤษเมื่อไม่มีคีย์เฉพาะ

## ข้อจำกัดในปัจจุบัน

- API ทางธุรกิจยังคงเป็นอะแดปเตอร์ภายในเบราว์เซอร์ และไม่ได้เชื่อมต่อกับแบ็กเอนด์เซิร์ฟเวอร์สำหรับโปรดักชัน
- โครงการใหม่จะสืบทอดโครงสร้างบางส่วนจากโครงการสาธิต แทนที่จะเริ่มจากแม่แบบที่ว่างเปล่าทั้งหมด
- ขั้นตอนการแก้ไขต้นทุนจริง สื่อ และบันทึกกิจกรรมยังไม่ได้เปิดใช้งานอย่างครบถ้วน
- รายละเอียดกลุ่มโครงการ การเพิกถอนการแชร์ และการควบคุมวันหมดอายุของลิงก์ยังต้องเชื่อมต่อเพิ่มเติม
- การรีเฟรชทั้งหน้าจะต้องกรอกรหัสผ่านอีกครั้งเพื่อปลดล็อกพื้นที่ทำงาน
- รวมการรองรับ PWA แล้ว แต่ยังไม่ได้กำหนดค่าการทดสอบ Lighthouse ขั้นตอนการติดตั้ง และการทดสอบออฟไลน์ตั้งแต่ต้นจนจบแบบอัตโนมัติ
- หน้าแบบไดนามิกที่ยังไม่ได้แคชและปลายทางเครือข่ายสดอาจไม่พร้อมใช้งานขณะออฟไลน์ หน้าสำรองเมื่อออฟไลน์และข้อมูลในเครื่องไม่สามารถใช้แทน API ระยะไกลได้
